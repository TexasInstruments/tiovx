How to build for x86 Win32 Platform

Requirements
  • GCC and Windows
    • Currently the build is tested with "mingw" that is installed as part of CodeBlocks IDE
    • Install "CodeBlocks IDE" 13.12 or higher [http://www.codeblocks.org/]
    • Make sure GNU make is installed and visible in your system path
      • GNU make "gmake" is available as part of XDC package from TI
  • Install the source package for VXLIB
    • IMPORTANT: To run VXLib in HOST emulation mode source package of VXLIB is required
Setup
  • Edit <TIOVX_PATH>/tools_path.mk
    • modify "GCC_WINDOWS_ROOT := C:/CodeBlocks/MinGW" to match your environment if needed
    • modify "VXLIB_PATH" to match the path where VXLIB package is installed
  • Edit <TIOVX_PATH>/Makefile
    • set BUILD_EMULATION_MODE=yes and BUILD_TARGET_MODE=no
Build
  • Open a command prompt at <TIOVX_PATH>
  • Type "gmake -s -j" to build TIOVX and associated libraries
    • This builds the TIOVX libaries and conformance test executable
    • This uses pre-built library for VXLIB from "<TIOVX_PATH>\lib\PC\X86\WINDOWS\<PROFILE>\*.a"
    • The VXLIB is compiled in host emulation mode for 32bit x86 windows systems
      • NOTE: MinGW GCC compiler supports only 32b compilation mode
  • Conformance test executable is output at
    • "<TIOVX_PATH>\out\PC\X86\WINDOWS\debug\vx_conformance_tests_exe.exe"
Running the conformance test
  • Open a command prompt at the folder "<TIOVX_PATH>\out\PC\X86\WINDOWS\<PROFILE>\"
  • Execute "vx_conformance_tests_exe.exe" to run the conformance test
  • The conformance test reads input files located in below folder
    • "<TIOVX_PATH>\conformance_tests\test_data"
    • The path specified relative to the "<TIOVX_PATH>\out\PC\X86\WINDOWS\<PROFILE>\" within the executable itself


How to build for x86 Linux Platform

Requirements
  • GCC and Linux
    • Currently the build is tested with Ubuntu 14.04
    • Linux machine needs to be a 64bit x86 machine
    • Make sure GCC v4.8.4 or higer is installed and visible in your system path

How to build for Processor SDK Vision Platform

Requirements
  • Install Vision SDK v2.12 or higher, this installs all packages need to compile and run TIOVX on TI platform.
Setup
  • NOTE: if TIOVX is located at folder "<VSDK_PATH>/ti_components/algorithms_codecs/tiovx" then below variables need not be modified
  • NOTE: Make sure to modify below variables under "Windows_NT" and/or "Linux" depending on the type pf machine that is used for build
  • Edit <TIOVX_PATH>/tools_path.mk, modify path for below variables to match your install environment
    • XDC_PATH
    • TIARMCGT_ROOT
    • GCC_ROOT
    • CGT6X_ROOT
    • ARP32CGT_ROOT
    • BIOS_PATH
    • VXLIB_PATH
    • PDK_PATH
    • VSDK_PATH
  • Edit <VSDK_PATH>/tools_path.mk and edit below variable to match your environment
    • "OpenVX_PATH ?= $(TI_SW_ROOT)/algorithms_codecs/tiovx"
  • Edit <TIOVX_PATH>/Makefile
    • set BUILD_EMULATION_MODE=no and BUILD_TARGET_MODE=yes
Build
  • Open a command prompt at <TIOVX_PATH>
  • Type "(g)make -s -j" to build TIOVX and associated libraries
    • This builds the TIOVX libaries
    • Conformance test is built as a library
  • The Vision SDK executable with the conformance test needs to be build from Vision SDK folder
    • Follow the Vision SDK user guide to build Vision SDK.
    • When Vision SDK is built, it will link against the tiovx and conformance test libraries.
    • NOTE:
      • Use MAKECONFIG=tda3xx_evm_bios_openvx when building Vision SDK
      • Only IPU1_0 and DSP1 are required to run the conformance test, other CPUs can be disabled from VSDK build if only conformance test will be run
Running the conformance test
  • Flash/copy the Vision SDK bootloader, "AppImage" and setup the EVM as mentioned in Vision SDK user guide
  • Copy the test data used in conformance test to an SD card
    • Copy all files and folders from "<TIOVX_PATH>\conformance_tests\test_data" to root folder in SD card
  • Boot the EVM, using the SD card and use the menu in UART terminal to navigate to the OpenVX use-cases and run the conformance test
