00007 ## \file vx_tutorial_graph_image_gradients_pytiovx_uc.py This file show example of using basic PyTIOVX APIs to
00008 #       describe an OpenVX graph.
00009 #
00010 #        To import PyTIOVX module so that the APIs can be called in this file add below line
00011 #        \code
00012 #        from tiovx import *
00013 #        \endcode
00014 #
00015 #        The basic steps one follows to describe the graph for PyTIOVX tool are,
00016 #        - Create a context object.
00017 #          - Pass as input a string which is used as a perfix to name the generated code and function.
00018 #        - Create a graph object object
00019 #        - Create data objects.
00020 #          - TIP: specify a name for data objects via name="xyz" to be able to identify it in the generated
00021 #            C code and image file
00022 #        - Create node objects and link it to required data objects
00023 #          - TIP: Add nodes to graph as you create them using graph.add()
00024 #          - TIP: Use target=Target.<TargetName> to specify the target on which this node runs.
00025 #        - Add graph to context using context.add()
00026 #        - Generate C code for the objects in a context using
00027 #          \code
00028 #          ExportCode(context).export()
00029 #          \endcode
00030 #        - Generate image file for the objects in a context using
00031 #          \code
00032 #          ExportImage(context).export()
00033 #          \endcode
00034 #        - Run the PyTIOVX on this file using
00035 #          \code
00036 #          python <.py file>
00037 #          \endcode
00038 #        - In case of any error reported by the PYTIOVX tool, correct the input .py file
00039 #        - Verify the generated image file to confirm the graph is as expected.
00040 #        - Write the remaining C code to invoke the generated APIs
